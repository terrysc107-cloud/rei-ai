# Phase 05 - Resource System Expansion

## Purpose

Make the resource system feel operational and valuable outside the lesson pages.

## Why This Phase Matters

Resources are where users start seeing business-ready assets, not just course content. This is a big part of what makes the product feel practically useful.

## Scope

- worksheets
- checklists
- SOPs
- templates
- resource collections

## Required Work

1. Expand the number and quality of resources.
2. Add more resources tied directly to tracks and lessons.
3. Create themed operator packs by use case.
4. Clarify what each resource helps the user accomplish.
5. Design how future downloadable assets will slot into the system.

## Deliverables

- stronger resource library
- resource packs by use case
- better relation between resources and lessons
- clearer operational value per item

## Dependencies

- track and lesson build-out should be mature enough to support tighter resource mapping

## Definition Of Done

- the resource library feels like a working toolkit
- users can identify which resource helps with which business problem
- resources add value even outside the core lesson flow

## Risks And Notes

- avoid decorative assets that do not support real business workflows
