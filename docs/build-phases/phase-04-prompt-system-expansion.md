# Phase 04 - Prompt System Expansion

## Purpose

Turn the prompt library from a useful section into a core product asset.

## Why This Phase Matters

Prompts are one of the product's main practical outputs. If they stay too small or too generic, the product will underdeliver on real-world usefulness.

## Scope

- prompt count
- prompt quality
- prompt grouping
- prompt variations
- prompt discoverability

## Required Work

1. Expand prompt count significantly across all major real estate workflows.
2. Add beginner and advanced versions where useful.
3. Add `when to use`, `when not to use`, and `best input format` guidance.
4. Create workflow packs that bundle prompts into repeatable sequences.
5. Improve prompt categorization and filtering.
6. Clarify what a saved prompt means in the context of building a reusable operating library.

## Deliverables

- expanded prompt inventory
- prompt packs by business use case
- better metadata and categorization
- stronger educational framing around prompt usage

## Dependencies

- curriculum tracks should be mostly complete so prompts map naturally to lessons

## Definition Of Done

- users can find prompts by problem, not just by title
- prompts feel like real tools, not just examples
- the library meaningfully supports the rest of the course

## Risks And Notes

- avoid bloating the prompt library with low-value variations
- quality and applicability matter more than raw count
