# Phase 10 - Content Studio Expansion

## Purpose

Build a stronger local content operations layer before real admin and backend tools exist.

## Why This Phase Matters

The product still needs a clean way to inspect curriculum, coverage, and gaps while it is file-backed. This phase improves maintainability during build-out.

## Scope

- curriculum inventory
- gap tracking
- editorial visibility
- quiz/resource/prompt coverage tracking

## Required Work

1. Expand the content studio beyond simple counts and lists.
2. Show completeness status for tracks, modules, lessons, prompts, and resources.
3. Surface curriculum gaps and editorial priorities.
4. Add better visibility into quiz coverage and prompt density.
5. Prepare the structure that a future true admin system can inherit.

## Deliverables

- stronger local content studio
- better editorial visibility
- easier curriculum management during the pre-backend phase

## Dependencies

- major content systems should already exist in enough depth to inspect meaningfully

## Definition Of Done

- the content studio helps guide build priorities
- it becomes easier to see what is incomplete
- it serves as a useful bridge to future admin tooling

## Risks And Notes

- avoid turning the local content studio into a fake CMS
- keep it high-signal and lightweight
