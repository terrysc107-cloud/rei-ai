# Phase 08 - Custom Build Funnel Completion

## Purpose

Complete the service upsell path so the app clearly supports both self-serve learning and done-with-you implementation.

## Why This Phase Matters

This is one of the product's strongest monetization angles. It needs to feel intentional and trustworthy, not like an extra form bolted onto the app.

## Scope

- offer positioning
- intake quality
- request examples
- qualification logic
- service framing

## Required Work

1. Tighten custom build positioning across the site.
2. Improve intake quality and expectations.
3. Add clearer examples by business problem and user type.
4. Show when the service path is a better fit than self-serve learning.
5. Define what the user receives from the request process.

## Deliverables

- stronger custom build offer
- better qualification flow
- more convincing transition from course to service

## Dependencies

- pricing and product positioning should already be reasonably mature

## Definition Of Done

- users can clearly tell whether the service path is for them
- requests are more qualified and specific
- the offer feels integrated into the product, not separate from it

## Risks And Notes

- vague service positioning weakens the product-business connection
