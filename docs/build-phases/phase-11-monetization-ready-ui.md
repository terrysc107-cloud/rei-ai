# Phase 11 - Monetization Ready UI

## Purpose

Make the UI visibly ready for paid tiers and premium paths before backend access control exists.

## Why This Phase Matters

The product should already communicate how monetization works before auth, gating, and billing are added.

## Scope

- locked states
- upgrade prompts
- premium previews
- tier language
- upsell framing

## Required Work

1. Design locked-state patterns for future gated content.
2. Add upgrade CTAs and premium-preview components.
3. Clarify which parts of the product are free, guided, or service-led.
4. Add tier-aware messaging to prompts, resources, and tracks where appropriate.
5. Tighten the connection between pricing and actual in-product behavior.

## Deliverables

- monetization-aware UI
- clearer free vs premium experience
- better product-business alignment

## Dependencies

- pricing and offer structure should already be reasonably clear

## Definition Of Done

- the business model is visible in the interface
- paid paths do not feel bolted on later
- the product has obvious upgrade points without breaking trust

## Risks And Notes

- avoid dark patterns or fake gates that frustrate users
