# Phase 01 - Core User Path Clarity

## Purpose

Make the product easy to understand and navigate so users know exactly what to do first, what to do next, and what outcome they are working toward.

## Why This Phase Matters

The app is visually much stronger than it was at the start, but users can still feel too many equal-weight choices. This phase reduces ambiguity and makes the golden path obvious.

## Scope

- landing page path clarity
- dashboard next-step hierarchy
- track selection guidance
- resume and continue behavior
- clearer completion cues

## Required Work

1. Define the single primary new-user journey.
2. Define the single primary returning-user journey.
3. Reduce dashboard competition between too many CTAs.
4. Add stronger `Start here`, `Continue`, `Next lesson`, and `Complete this track` UI states.
5. Clarify what lesson completion, track completion, and project completion mean.

## Deliverables

- a documented golden path through the app
- one primary CTA per major screen
- clearer onboarding copy
- clearer resume logic
- stronger progress language across tracks and lessons

## Dependencies

- current landing page, dashboard, start page, track pages

## Definition Of Done

- a first-time user can explain where to start within a few seconds
- a returning user can resume work without exploring the nav
- every major page communicates one primary next action
- the product no longer feels visually polished but directionally vague

## Risks And Notes

- adding more content without path clarity will increase confusion
- do not solve this with more cards or more links; solve it with stronger hierarchy
