## Product Definition Of Complete

The app should be considered complete when:

- a new user can land on the site, understand the offer, and choose a clear next step
- a learner can sign up, choose a track, complete lessons, save prompts, take quizzes, and submit a final project without confusion
- the course content feels fully written rather than scaffolded
- the prompt, resource, and workflow systems feel like reusable business assets
- the service upsell is clearly positioned and naturally connected to the learning product
- persistence, auth, access control, and production reliability are in place

## How To Use These Phase Docs

Each phase file includes:

- purpose
- why the phase matters
- scope
- deliverables
- dependencies
- definition of done
- risks and notes

The intended workflow is to complete one phase at a time and only advance when the definition of done is clearly met.
